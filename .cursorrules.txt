# ARIA — Personal AI Chief of Staff



## Project Overview

ARIA is a Telegram bot that acts as Sumedh's personal AI chief of staff. It manages his calendar and tasks via Reclaim.ai, interprets natural language commands, and takes actions on his behalf.

# Part 1: ARIA Feature / Logic Map (Target Spec)

## 1. Master flow

```mermaid
flowchart TD
    msg[User message] --> tier1{Tier-1 keyword?}
    tier1 -->|undo| undoKw[Keyword undo rollback]
    tier1 -->|snooze| snoozeKw[Snooze helper → reschedule_task]
    tier1 -->|no| llm[LLM → JSON]

    llm --> intent{action_required?}
    intent -->|false| replyOnly[Reply only, no buffer]
    intent -->|true| missing{All params known?}
    missing -->|no| clarifyLoop["clarification_required=true\nAsk ALL missing fields\nconversation_buffer ON\nwhile loop until complete"]
    missing -->|yes| classify{Compound action?}

    classify -->|yes| defer[Hold 30s deferred buffer]
    classify -->|no| execNow[Execute immediately]
    defer --> execNow2[Execute after buffer unless amended]
    execNow --> dispatch[dispatch]
    execNow2 --> dispatch

    dispatch --> confirm[Append confirmation template]
    confirm --> ledger[Store last action 30s for amend/undo]
```

### Core rules (your G4 + clarification model)

| State | `action_required` | `clarification_required` | `conversation_buffer` |
|-------|-------------------|--------------------------|------------------------|
| Vague / no intent (#7, #8) | `false` | `false` | OFF |
| Wants action, missing params | `false` (until complete) | `true` | ON |
| All params gathered | `true` | `false` | OFF (after execute) |

**Clarification loop:** like a `while not all_params_gathered()` — each turn asks for **every** still-missing field in one message; merge user reply; repeat until the LLM can emit a complete call.

**Hardcoded keywords only:** `undo` (+ cancel variants), `snooze`. Everything else (including amend) = **LLM + last-action context**.

## 2. Shared state

| Store | Shape | Refresh | Contents |
|-------|-------|---------|----------|
| **`week_event_cache`** | `dict[event_id → {title, start, end, task_id, …}]` + chronological index | **GCal webhook** | **Reclaim task blocks only** (not lunch/commute) |
| **`current_task`** | ping context for active block | block-start ping | title, event_id, times, is_ongoing |
| **`conversation_buffer`** | chat turns | append during clarify | only when clarification active |
| **`last_action_ledger`** | fn + params + touched entities | on every execute | **30s TTL**, injected into system prompt for amend |
| **`pending_deferred`** | queued compound calls | on classify | 30s hold before dispatch |

**“Last task”** = chronologically prior Reclaim task event in `week_event_cache` (same as “previous block” for ping scenarios).

## 3. Tier-1 bypasses (no LLM)

### `undo`
- Keyword detector → rollback last action within **30s**
- No LLM

### `snooze`
- Pattern: `"snooze {task} for {duration}"` OR `"snooze {task} until {time}"`
- Dedicated **`parse_snooze_spec(message)`** helper:
  - `for X hours` → relative from now
  - `until Y` → absolute via existing time parser
- Always → **`reschedule_task`** (Reclaim snooze, never missed-work move)

## 4. LLM-callable functions

### Primitives (simple → execute now, 30s undo)

| Function | When | Key params |
|----------|------|------------|
| **`create_task`** | New task after disambiguation | title, due, duration, category WORK/PERSONAL |
| **`create_event`** | Lunch, commute, evening busy block | name, start, end (natural) |
| **`complete_task`** | User finished / wants off schedule | task_query |
| **`extend_task_total`** | Whole-task budget | task_query, additional_time |
| **`extend_task_instance`** | Reclaim task block extension | task_query, additional_time |
| **`extend_current_gcal_block`** | Non-task GCal block (lunch/commute) | event_id or implicit current, end/until |
| **`reschedule_task`** | Snooze (Tier-1 or LLM) | task_id, snooze_until |
| **`move_due_date`** | **New** — move due only | task_query, due_date_natural |
| **`update_task`** | Patch other fields | task_query, patches |
| **`get_schedule_for_window`** | Read-only (#17) | day, period (morning/afternoon/…) |

### Composites (compound → **30s deferred buffer** before execute)

| Function | When | Notes |
|----------|------|-------|
| **`switch_active_task`** | User on a **different** task during ping | 1) Snooze pinged task **1 hour**; 2) **`plan_work(new, now, duration)`** |
| **`resume_previous_task`** | Still on **previous** task, new ping fired | 1) Snooze pinged task **1 hour**; 2) **`plan_work(previous, now, duration)`** or `extend_task_instance` if extending same block |
| **`reschedule_missed_work`** | Missed/skipped today’s work | Find today’s blocks from cache; default target **+1 hour from now**; clarify if multiple blocks |

**Ping mismatch rule:** not working on the pinged Reclaim task → **snooze it 1h** (`FROM_NOW_1H` / 4 chunks), then handle what user *is* doing. Do **not** GCal-bump the pinged block (auto-locks).

### Switch vs extend vs resume

| Intent | Same as ping? | Action |
|--------|---------------|--------|
| Extend ("30 more min on this") | Yes | `extend_task_instance` or `extend_current_gcal_block` |
| Switch ("doing orgo instead") | No | `switch_active_task` |
| Resume ("still on last task") | Previous task | `resume_previous_task` |

### Reclaim “start now” (tasks)

- **`startNow`** in swagger is **habits only** — not for task blocks.
- **Tasks:** `POST /api/planner/plan-work/task/{id}?dateTime=now&durationMinutes=…` is the **primary** schedule-now path (switch/resume step 2). `POST /api/planner/start/task/{id}` is optional timer start. `reschedule_task` / snooze endpoints are for **snoozing away** the pinged task or explicit user snooze — not for “work on X now”.

|--------|-------------------------------|---------|

| **`parse_duration(text, convert_to_chunks=False)`** | duration_parser / cal_helper | `"30 minutes"` → 30; `"until 5pm"` → minutes-from-now; optional chunk conversion |

| **`parse_upcoming_time(phrase, now)`** | inference/cal_helper | No am/pm → next occurrence (#18, #20): 11AM + "12:30" → PM; 11PM + "12:30" → AM |
| **`find_task_blocks(task_query, day)`** | cal_helper | Today’s (or week’s) blocks for a task from cache (#9, #10) |
| **`get_events_for_window(day, period)`** | queries/cal_helper | Overlap filter with your bands (#17); **includes all GCal events** |
| **`compute_missing_fields(intent, partial)`** | clarification | Returns list of still-missing params for while-loop |
| **`merge_clarification_reply(pending, message)`** | clarification | Merge user answer into partial_params |

| **`build_action_confirmation(fn, params, result)`** | cal_helper or dispatch | Template: `"I've created a lunch block from now until 12PM"` (#21) |
| **`find_min_safe_break(...)`** | buffer_analysis | When unsafe, compute smallest safe break (#19b) |

## 7. Action tiers (G3)

| Tier | Execute | Undo mechanism | Examples |
|------|---------|----------------|----------|
| **Simple** | Immediately | 30s ledger + rollback | complete, create_event, extend_instance, snooze, create_task, move_due_date |
| **Compound** | **After 30s buffer** (user can amend in-window) | Prefer cancel/amend before fire; else rollback | switch_active_task, resume_previous_task, possibly multi-block reschedule |

---

## 8. Confirmation & amend

- **Every successful action:** deterministic confirmation template (not left to LLM prose alone).
- **Amend (30s):** exact last `{function, params}` in system prompt; LLM decides if new message amends it.
- **Literal `undo`:** keyword, no LLM.
- **Natural correction (`i meant X`):** two-step — rollback first JSON/turn, then redo with corrected params.
- **Snapshots:** you want to revisit; spec leans on **ledger + 30s** rather than heavy pre-snapshotting for simple actions.
