# ARIA — Personal AI Chief of Staff



## Project Overview

ARIA is a Telegram bot that acts as Sumedh's personal AI chief of staff. It manages his calendar and tasks via Reclaim.ai, interprets natural language commands, and takes actions on his behalf.



## Stack

- **Runtime**: Python 3.14, FastAPI, Uvicorn

- **Bot**: python-telegram-bot (async, polling mode)

- **Scheduler**: APScheduler (AsyncIOScheduler)

- **LLM**: Groq (llama-3.3-70b-versatile for reasoning, whisper-large-v3-turbo for voice)

- **Embeddings**: Google Gemini Embedding 2 (cosine similarity for task resolution)

- **Calendar**: Reclaim.ai REST API (httpx async) + Google Calendar API (via service account)

- **Webhooks**: Google Calendar push notifications via ngrok tunnel (auto-renewed every 6 days)



## File Structure

- `main.py` — FastAPI app, lifespan startup/shutdown

- `bot.py` — Telegram handlers, scheduler pings, prep_next_block

- `inference.py` — LLM calls, date/duration parsing, bool normalization, validation

- `system_prompts.py` — SYSTEM_PROMPT_CAL (task_query-based, no full task list)

- `cal_helper.py` — Task map, cosine similarity resolution, async dispatch

- `reclaim.py` — Async Reclaim.ai and Google Calendar API calls

- `gcal.py` — GCal get/move/delete/create with rollback snapshots
- `intent.py` — Tier-1 substring + tier-2 difflib intent detection
- `queries.py` — Per-turn read-only schedule snapshots (weekly, missed work)
- `composites.py` — Multi-step ops: reschedule_missed_work, switch_active_task, extend_current_block, ping router
- `rollback.py` — Action ledger, undo, and 5-minute amendment window
- `webhooks.py` — /gcal-webhook endpoint, debounce, watch registration/renewal

- `config.py` — API keys and constants
- `duration_parser.py` — natural-language duration → minutes/chunks



## Key Patterns

- Be deterministic wherever possible, so long as outcomes are maximized in accuracy

- Everything is async — Reclaim uses httpx AsyncClient; GCal sync calls use asyncio.to_thread

- LLM returns JSON with fields: audit, action_required, clarification_required, reply, calls

- LLM never sees the full task list — per-turn snapshots via queries.py for read questions; task_query for writes

- current_task context is injected into the system prompt for ping follow-ups

- dispatch() resolves task_query → task_id via get_task_by_query() before calling Reclaim
- Composites return {ok, summary, snapshots}; execute_calls() in rollback.py handles undo ledger
- Ping responses (extend/switch) and missed_work route deterministically before LLM executes writes
- Undo/cancel intercept runs before LLM; within 5 min, corrections amend in place via update_task (no auto-undo)
- create_task requires eventCategory (WORK/PERSONAL); bot always asks work or personal before creating
- Reclaim GET responses cached briefly (8s); invalidated on writes and GCal webhooks

- Dates passed as natural language and converted to ISO 8601 via parse_to_iso()
- Durations passed in *_natural fields and converted via duration_parser.py — LLM may also pass numeric chunks/minutes; inference normalizes before dispatch

- conversation_buffer accumulates messages only during clarification loops

- Task map rebuilds after write operations and on webhook fire



## Coding Conventions

- Print statements for all API successes/failures with ✅/❌ prefix

- Timeout/NetworkError retries on all Telegram send calls (send_with_retry helper)

- Module-level globals for scheduler, bot_app, current_task, conversation_buffer
